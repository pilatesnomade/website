# Plan: Mailjet Contact Form Integration

## Overview
Add a contact form in Footer.astro that integrates with Mailjet API via a Vercel serverless function to:
1. Create a contact via `/v3/REST/contact`
2. Add the contact to a mailing list via `/v3/REST/listrecipient`

## Implementation Steps

### 1. Create Vercel Serverless Function
**File:** `src/pages/api/subscribe.ts`

The function will:
- Accept POST requests with `email` in the body
- Use `MAILJET_API_KEY` and `MAILJET_API_SECRET` environment variables for authentication
- Use Basic Auth with base64-encoded credentials
- First call: Create contact at `https://api.mailjet.com/v3/REST/contact`
- Second call: Add contact to list at `https://api.mailjet.com/v3/REST/listrecipient`
- Return appropriate JSON responses and status codes

### 2. Update Footer.astro Form
**File:** `src/layouts/Footer.astro`

Changes:
- Update form `action` from `#` to `/api/subscribe`
- Add client-side JavaScript for:
  - Form submission via fetch (AJAX)
  - Loading state on button
  - Success/error feedback messages
  - Form reset on success

### 3. Add Environment Variables
**File:** `.env` (local) and Vercel dashboard (production)

Required variables:
- `MAILJET_API_KEY` - Mailjet API key
- `MAILJET_API_SECRET` - Mailjet API secret
- `MAILJET_LIST_ID` - Target contact list ID (for listrecipient call)

### 4. Create .env.example
**File:** `.env.example`

Document required environment variables for other developers.

## Technical Details

### API Response Handling
- If contact already exists (409 Conflict), still add to list
- Handle API errors gracefully with user-friendly messages
- Return `{ success: true }` on success, `{ error: "message" }` on failure

### Form Validation
- HTML5 email validation (built-in)
- Client-side required field check
- Server-side validation of email format

### Security Considerations
- API credentials stored in environment variables (never exposed to client)
- Server-side validation
- CORS handled by same-origin request
